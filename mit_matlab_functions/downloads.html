Matlab tools for network analysis


Nov 5: This upload is in progress. For questions, comments, suggestions and corrections, please email.

This toolbox was written about three years ago (2006) and randomly updated, because I switched from Matlab to Python due to the growing size of the datasets I had to analyze. Some of the Python routines do not have an equivalent in Matlab, but the Python code will also published here for reference. These routines are useful for someone who wants to start hands-on work with networks fairly quickly, and explore simple graph statistics, distributions, simple visualization and other common network theory metrics. This code is not object-oriented, and should be easy to use, read and improve upon. There is some overlap with the grTheory toolbox published on Matlab Central, however our routines are not tailored to pure graph theorists but more to applied graph (network) theory research. No code is perfect so please contact me if you notice errors or have pointers to better routines.

The code is classified by functionality as follows:

Basic network routines
          getnodes.m
          getedges.m
          numnodes.m - number of vertices/nodes in the network;
          numedges.m - number of edges/links in the network;
          link_density.m - the density of links of the graph;
          selfloops.m - number of selfloops, i.e. nodes connected to themselves;
          multiedges.m - number of arcs (i,j) with multiple edges across them;
          average_degree.m - the average degree (# links) across all nodes;
          num_conn_comp.m - the number of connected components using the algebraic connectivity;
          find_conn_comp.m - the number of connected components in the graph;
          giant_component.m - extract the giant component only;
          graph_complement.m - the complement graph;
          graph_dual.m - the graph dual (or line graph, adjoint graph);
          subgraph.m - return the subgraph given the supergraph adj matrix and the set of subgraph nodes;

Diagnostics
          issimple.m - check whether the graph has selfloops and multiple edges;
          isdirected.m - directed or undirected graph (right now uses issymmetric);
          issymmetric.m - check whether a matrix is symmetric;
          isconnected.m - check whether a graph is connected;
          isweighted.m - determine whether the graphs has weighted links;
          isregular.m - check whether it's a regular graph;
          iscomplete.m - check whether the graph is complete;
          iseulerian.m - find out whether it's an eulerian graph;
          istree.m - check whether the graph is a tree;
          diagnose_powerlaw.m - diagnose a sequence as power law or not;

Conversion routines - see page 34, Table 1.1
          adj2adjL.m, (adjL2adj.m) - convert an adjacency matrix to an adjacency list;
          adj2edgeL.m, (edgeL2adj.m) - convert an adjacency matrix to an edge list;
          adj2inc.m, (inc2adj.m) - convert adjacency matrix to incidence matrix and vice-versa;
          adj2str.m, (str2adj.m) - convert adjacency matrix to a string graph representation;
          adjL2edgeL.m, (edgeL2adjL.m) - convert adjacency list to edge list;
          adj2simple.m - Remove selfloops and multiedges from an adjacency matrix
          edgeL2simple.m - remove selfloops and multiedges from an edgelist
          add_edges.m - adding edges that occur multiple times in an edgelist; summing weights;

Centrality measures & Distributions
          degrees.m - returns the degree, in-degree and out-degree sequence of an arbitrary adjacency matrix;
          ave_neighbor_deg.m - computes the average degree of neighboring nodes for every verte;
          closeness.m - computes the closeness centrality for all vertices;
          node_betweenness.m - a slow node betweenness algorithm, based on the # of shortest paths definition;
          edge_betweenness.m - a slow edge betweenness algorithm, based on the # of shortest paths definition;
          node/edge betweenness using dijkstra;
          eigencentrality.m - node eigencentrality (eigenvector corresponding to the largest eigenvalue);
          clust_coeff.m - compute two clustering coefficients, based on triangle loops and local clustering;
          pearson.m - pearson degree correlation;
          rich_club_metric.m
          s_metric.m - the sum of products of nodal degrees across all edges;

Spectral properties
          laplacian_matrix.m - the Laplacian of the graph: degree matrix minus the adjacency;
          graph_spectrum.m - the sorted eigenvalues of the Laplacian;
          algebraic_connectivity.m - the second smallest eigenvalue of the Laplacian;
          fiedler_vector.m - the eigenvector corresponding to the algebraic connectivity;
          eigencentrality.m - see Centrality measures & Distributions

Distances
          simple_dijkstra.m - simple implementation of Dijkstra's algorithm, does not remember the path;
          Dijkstra with storing the paths;
          shortest_pathDP.m - shortest path algorithm using dynamic programming;
          kneighbors.m - returns the indices of all neighbors k links away (for a given node);
          diameter.m - the longest shortest path in the graph;
          ave_path_length.m - the average shortest path in the graph;
          distance_distribution.m - the # of pairs of nodes at a distance x, divided by the total # of pairs;

Comparing graphs
          aligned graph hamming distance;
          hubs and authorities graph similarity (Blondel);

Modularity
          bi-partitioning;
          clustering;
          Newman-Girvan algorithm;
          Newman eigenvector modularity;

Motifs
          loops3.m - count all loops of size 3 in the graph;
          loops of size 4;
          find all cliques;
          general heuristic motif-finding routine;

Building graphs
          random_graph.m - generate a random graph adjacency matrix using various models;
          random_graph(n) - Erdos-Renyi graph with n nodes and probability of attachment 0.5;
          random_graph(n,p) - Erdos-Renyi graph with n nodes and probability of attachment p;
          random_graph(n,[ ],E) - random graph with E number of edges;
          random_graph(n,[ ],[ ],distribution,@fun) - nodal degrees are drawn from a particular distribution
                                                  (uniform, normal, geometric, binomial, exponential or custom);
          random_graph(n,[ ],[ ],'sequence',[ ],deg_seq) - the nodal degrees match a given degree sequence;
          random_modular_graph.m - random graph with modules, given # nodes, # modules, and densities;
          random graph with controlled zero clustering coefficient and zero degree correlation;
          canonical nets;
          PriceModel.m - Price's citations growth model;
          preferential_attachment.m - preferential attachment graph;
          s-max graph;
          exponential growth;
          master equation;
          Newman-Gastner spatial distribution model;
          Fabrikant model;
          randomized hierarchies;
          ...

Plotting distributions
          pdf_cdf_rank.m - pdf, cdf and rank distributions of a given sequence;

Drawing graphs
          adj2pajek.m, (pajek2adj.m) - convert an adjacency matrix to a Pajek readable format: output .net file;
          edgeL2pajek.m, (pajek2edgeL.m) - convert an edge list to a Pajek readable format: output .net file;
          pajek2xyz.m - Extract node x,y,z coordinates from a Pajek .net file, useful for drawing in Matlab;
          draw_circ_graph.m - draw a graph with all nodes in a circle, ordered by degree;
          draw_dot_matrix.m - dot matrix plots with sorted columns/rows by various measures;
          el2geom.m - plot an edgelist geographically with color-coding of edge weights; need coordinates;
          Kamada-Kawai algorithm;
          Harel algorithm;

Auxiliary
          symmetrize.m - symmetrize a matrix;
          symmetrize_edgeL.m - symmetrize an edge list (make it undirected);
          ind2ij.m - convert array index to matrix (i,j) coordinates;
          ij2ind.m - convert (i,j) matrix coordinates to array index;
          num_conn_triples.m - the number of connected triples of nodes, given the adjacency matrix;
          purge.m - remove elements from a list, while preserving the order;
          random_number.m - random number generator using the computer clock;
          min_span_tree.m - construct a minimum spanning tree given the adjacency matrix;
          BFS.m - breadth-first tree;
          DFS.m - depth-first tree;
          simulated annealing tool;
          unitize.m - convert a vector to a unit vector;

Chinese airlines 2006 timetable data



This data is publicly available (or was in 7/2006) and was collected for the purposes of a study on airline network topology and robustness. The entire timetable as collected, is linked in "Raw data" below. The entire edgelist generated from it is linked in "Master edgelist". It contains all origin-destination pairs, including airline code, aircraft type and number of weekly departures ([airline, origin, destination, a/c type, frequency]). The origin-destination pairs and the aircraft types were used to create networks for each airline, and study the network capacity in terms of seats offered every week. The analysis of 20 Chinese airline networks is described in the material linked on the left.
